# 投影仪评论分析配置文件检查报告

## 一、代码级问题

### 1.1 正则表达式错误
**位置**: 第78行 `prefix_pattern`
```json
"prefix_pattern": "^(事件:|退货:|事故:|返回:|回国:|回报:|问题描述:|客户反馈:|投诉:|投诉内容:|具体问题:|详细描述:)+"
```

**问题**: 
- 最后的 `+` 量词表示"一次或多次",但实际上应该是 `*`(零次或多次)
- 这会导致不匹配前缀的正常文本被错误处理

**修复**:
```json
"prefix_pattern": "^(事件:|退货:|事故:|返回:|回国:|回报:|问题描述:|客户反馈:|投诉:|投诉内容:|具体问题:|详细描述:)*"
```

### 1.2 字段命名不清晰
**位置**: `en_zh_translation_map`

**问题**: 
- 名称暗示"英中翻译映射",但实际是"英文关键词→中文标签"映射
- 容易造成理解偏差

**建议改名**:
```json
"en_keyword_to_label_map" // 或 "english_trigger_keywords"
```

### 1.3 列名重复问题
**位置**: column_settings
```json
"rating_columns": ["星级", "星级.1"]
```

**问题**: "星级.1" 是Excel自动生成的重复列名,应该检查原始数据源

**建议**: 规范化为 `["星级_原始", "星级_标准化"]` 或 `["用户星级", "系统星级"]`

## 二、语义规则问题

### 2.1 近义词冗余
**问题**: 大量近义词映射到相同标签,造成规则臃肿

**示例**:
```json
"清晰度/分辨率": {
    "模糊": "模糊",
    "不清晰": "模糊",  // 重复
    "不够清晰": "模糊", // 重复
    "不清楚": "不清楚",  // 应该映射到 "模糊"
    "发虚": "发虚",      // 应该映射到 "模糊"
    "糊": "糊"           // 应该映射到 "模糊"
}
```

**优化方案**: 统一映射到标准化标签
```json
"清晰度/分辨率": {
    "模糊": "模糊",
    "不清晰": "模糊",
    "不够清晰": "模糊",
    "不清楚": "模糊",
    "发虚": "模糊",
    "糊": "模糊",
    "看不清": "模糊",
    "边缘糊": "边缘模糊",  // 更具体
    "发虚严重": "模糊_严重" // 带程度
}
```

### 2.2 缺少否定词处理
**问题**: 没有处理"不"、"没有"等否定词

**风险示例**:
- "不模糊" → 错误匹配到"模糊"
- "没有噪音" → 错误匹配到"噪音"
- "不够亮" → 可能漏匹配"亮度不足"

**建议新增规则**:
```json
"negation_rules": {
    "pattern": "(不|没|没有|无|别|不要)+(太)?(.*)",
    "action": "reverse_polarity",  // 反转极性
    "examples": {
        "不模糊": "清晰",
        "没有噪音": "安静",
        "不够亮": "亮度不足"
    }
}
```

### 2.3 缺少程度词增强
**问题**: 缺少对"非常"、"特别"、"极其"等程度词的处理

**建议新增**:
```json
"intensity_modifiers": {
    "强增强": ["非常", "特别", "极其", "超级", "太", "相当", "十分"],
    "中增强": ["比较", "还算", "挺", "蛮"],
    "弱化": ["有点", "稍微", "略微", "稍"],
    "极端": ["完全", "彻底", "简直", "根本"],
    "weight_multipliers": {
        "强增强": 1.5,
        "中增强": 1.2,
        "弱化": 0.8,
        "极端": 2.0
    }
}
```

### 2.4 缺少上下文感知规则
**问题**: 某些词汇在不同上下文中含义不同

**示例**:
- "够亮"(正面) vs "太亮了刺眼"(负面)
- "很艳"(可能正面) vs "色彩失真"(负面)

**建议新增上下文规则**:
```json
"context_aware_rules": {
    "亮度": {
        "positive_context": ["够", "非常", "很", "挺"],
        "negative_context": ["太", "过于", "刺眼", "晃眼"],
        "threshold_keywords": {
            "亮": {
                "with_positive": "亮度满意",
                "with_negative": "太亮了刺眼",
                "standalone": "亮度满意"
            }
        }
    }
}
```

## 四、权重系统优化

### 4.1 权重应用不一致
**问题**: 
- `keyword_rules` 中有权重系统
- `exact_rules` 中没有权重
- 缺少组合权重计算规则

**建议统一权重系统**:
```json

    "base_weights": {
        "exact_match": 1.0,
        "keyword_match": 0.8,
        "regex_match": 0.9,
        "fuzzy_match": 0.6
    },
    "category_weights": {
        "亮度": 1.2,           // 核心维度加权
        "色彩": 1.2,
        "清晰度/分辨率": 1.3,  // 最重要维度
        "性价比": 1.1,
        "售后体验": 1.0,
        "外观与便携": 0.9      // 次要维度
    },
    "sentiment_weights": {
        "正面": 1.0,
        "负面": 1.2,           // 负面评价更重要
        "中性": 0.8
    },
    "combination_rules": {
        "multiple_positive": "max",  // 多个正面取最高
        "multiple_negative": "sum",  // 多个负面累加

    }
}
```
